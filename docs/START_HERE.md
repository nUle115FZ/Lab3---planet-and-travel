## 💡 Ключевые моменты

### 1. Приватное наследование
```cpp
class PriorityQueue : private DynamicArray<pair<T, K>>
```
**Зачем?** Скрыть методы Sequence, оставить только Enqueue/Dequeue.

### 2. Сложность Дейкстры
```
O((V + E) log V)
```
**Почему log V?** Операции с PriorityQueue (min-heap).

### 3. Формула веса
```cpp
totalCost = distance × (1 + riskFactor)
```
**Пример:** 100 св.лет, риск 0.3 → стоимость = 100 × 1.3 = 130

## 🎓 Вопросы для защиты

**Q: Почему PriorityQueue наследуется приватно?**  
A: Чтобы скрыть методы Sequence и предоставить только интерфейс очереди.

**Q: Какая сложность алгоритма Дейкстры?**  
A: O((V + E) log V) с PriorityQueue, O(V²) без неё.

**Q: Почему список смежности, а не матрица?**  
A: Эффективнее для разреженных графов: O(V+E) vs O(V²) памяти.

**Q: Как работает min-heap?**  
A: Бинарное дерево, родитель < детей. heapifyUp/Down за O(log n).

**Q: Можно ли Дейкстру для отрицательных весов?**  
A: Нет! Нужен Беллман-Форд.
